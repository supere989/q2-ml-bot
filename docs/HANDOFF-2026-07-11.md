   live deployment could never escape it (needed the harness's UDP reply
   already buffered at the exact instant checked), so every action applied
   as a zero fallback forever. Training's 48-bot batched timing gave enough
   incidental slack to mostly avoid it, which is why this went unnoticed

## Gotchas (each cost real time to discover â€” don't rediscover them)

- **`Q2_EXT_OBS=1` is required for every aim BC/canary artifact listed
  above** (219-dim obs; `encoder.rest.0.weight` has 147 inputs). The currently
  running unrelated main PPO was launched without it and is 209-dim. Do not
  infer shape from the step number; inspect the checkpoint or run
  `tools/verify_protocol.py`. A wrong setting fails with a size mismatch.
- **`ml2sk1` botlist (`n_bots=2`) doesn't spawn anything** on the WSL box â€”
  no error, server just sits idle, harness times out waiting for obs. Root
  cause not investigated. Use `n_bots=4`+ sizing for anything needing
  multiple bot slots (including multi-ML-bot setups like `ml_vs_ml.py`).
- **`fire_logits` is not a plain key** in `Q2BotPolicy.forward()`'s output
  â€” it's conditioned on the chosen weapon via
  `fire_logits_for(feat, weapon_idx)`. If you see `KeyError: 'fire_logits'`
  anywhere else, same fix pattern applies.
- **`q2ded` dlopens `lithium/game.so`**, not `gamex86_64.so` â€” always copy
  the build output to that exact filename. Servers reload it on a later
  round/map start, so copying into the active runtime *does* silently change
  a running trainer's semantics.
- **The C Makefile does not track header dependencies.** After changing
  `botstr.h` or another shared layout, use `make clean && make -j4`. Plain
  `make` after adding the terminal flag linked mixed `zgcl_t` offsets and the
  isolated server crashed in its first frame.
- **Never build/test C changes directly against `~/q2_lithium_merge`** while
  the trainer is running â€” its `q2ded` servers share that exact path and
  will pick up whatever's there on their next round rotation. Use a fresh
  scratch copy such as `~/q2_lithium_merge_scratch` with its own `Q2_ROOT`
  and port slab. Do not reuse the deprecated pre-fix runtime for experiments.
- **`~/q2-ml-bot` on WSL is an operational tree with no `.git`;** the local
  procreator clone/GitHub is canonical. Copy source files surgically and
  verify hashes; never overwrite its gitignored checkpoints/runs wholesale.
- **Observation ticks reset downward on map load.** Python freshness checks
  must not use `<`/`<=`. Exact same-tick replays are stale, except that a
  same-tick terminal following an already-returned nonterminal is a valid
  late death boundary and must be promoted. Same-new-tick packets first seen
  together may also contain distinct reward deltas and must remain mergeable.
- **A training `--seed` alone is not a paired game experiment.** Use
  `--seed`, `--game_seed`, and `--deterministic 1` together in lockstep mode.
  Old `use_mapqueue=0` runs inherited libc's default `rand()` stream rather
  than a time seed, but they lacked named independent game replicas and their
  threaded spatial RNG draws were scheduling-dependent. `use_mapqueue=1`
  has its own shuffle; C commit `8ddd53d` preserves `ml_game_seed` there too.
- **The recurrent anchor is intentionally constrained.** A positive
  `--aim_anchor_coef` requires `Q2_POLICY_STATEFUL=1` and one full-rollout
  minibatch (`batch_size >= n_steps * total_venvs`; 512 in the canaries).
  This keeps visible/fire masks globally weighted. The default is zero and a
  disabled seed-5301 reproduction was byte-identical to the old control.
- **`ML_PackObs()` still clears reward accumulators before UDP send success.**
  Terminal flags now retry after a failed send, but that frame's reward payload
  would be lost. Loopback `sendto` failures were not observed in the gates;
  fixing this correctly needs pack/commit separation and is a lower-priority
  protocol hardening item.
- **`__DATE__`/`__TIME__` are embedded in `game.so`** (version strings in
  `g_save.c`/`lithium.c`) â€” identical source produces different checksums
  on every rebuild. Don't use checksum equality to verify "did my change
  actually apply" â€” diff the disassembly (`objdump -d`) or the source
  instead.
- **Python subprocess stdout is fully buffered when redirected to a file**
  â€” always use `python3 -u` for anything backgrounded with `> log.txt`, or
  you'll see zero output until the process exits.
- **`pkill -f <pattern>` can self-match** if your own shell's command line
  (e.g. a later `grep` in the same compound command) contains the same
  pattern string â€” kills the wrapper instead of the target, surfaces as a
  confusing SSH exit 255. Keep pkill patterns isolated to their own
  command, or use the `[x]pattern` bracket trick.
- **A crashed older trainer can orphan q2ded children.** Current `train/ppo.py`
  registers owned-server cleanup at interpreter exit, but `SIGKILL` cannot run
  it. Before reusing an ablation port slab, inspect exact executable paths and
  UDP ports; kill only the isolated PIDs, never the active runtime's q2ded
  processes.
